## Rules

### 1. Do not chain git commands with `&&`

The dominant friction signal this week — 73 hook reminders across 57
sessions — is git command chaining. The hook explanation:

> Chaining git commands with '&&' can cause index.lock race conditions.
> The lock file from an index-modifying command (add, commit, rm, mv,
> reset) may not be released before the next command runs.

**Do**: issue each git call as a separate Bash invocation.

```
# Correct
git add file.ts
git commit -m "fix(scope): summary"

# Incorrect — triggers the hook
git add file.ts && git commit -m "fix(scope): summary"
```

This applies to *any* git command pair where the first one writes to
`.git/`. Read-only chains (`git status && git diff`) are not flagged
but should also be split for parallelism.

### 2. Reach for Glob before `find`

Combined volume of `find`-related friction (the reminder + the
parallel-call cancellations that follow it): **88 events across ~36
sessions**. The cancelled-parallel-find signal is particularly costly
because the agent typically fires several `find` calls in parallel,
all of which are killed at once.

**Do**:

```
# Correct
Glob(pattern="**/*.md")

# Incorrect — fires the hook AND wastes the parallel slot
Bash("find . -name '*.md'")
```

Use `find` only when you genuinely need flags Glob lacks: `-maxdepth`,
`-type d`, `-mtime`, `-print0`, etc.

### 3. Prefer Read over `cat`/`head`/`tail` for files

17 events across 16 sessions. The Read tool returns line-numbered
content and respects token budgets. Use `offset`/`limit` instead of
`head -N` / `tail -N`.

```
# Correct
Read(file_path="/abs/path", offset=100, limit=50)

# Incorrect
Bash("head -150 /abs/path | tail -50")
```

`cat` for non-file purposes (heredoc piping into a command) is fine.

### 4. Use TaskOutput for background-task output

When a Task is running in the background, do not `cat` its output
file. Use the TaskOutput tool with the task ID. 10 reminders across 9
sessions.

### 5. Do not chain `sleep` with another command

```
# Incorrect — blocked by the hook
sleep 30 && gh pr checks 123

# Correct — until-loop via Monitor, or run in background
until gh pr checks 123 | grep -q completed; do sleep 5; done
```

### 6. Keep pipelines under 5 pipes

8 sessions hit the >4-pipe reminder (one command had 8 pipes). Either:
ask the source command for JSON and pipe to `jq`, or use a single
`awk` for multi-step text work.

### 7. Do not enter plan mode for conceptual questions

`ExitPlanMode` was invoked 30 times in the window, often for prompts
that were questions or comparisons rather than change requests. Plan
mode is for *modifying* the codebase. For "how does X work", "what's
the difference between A and B", "explain Y", answer inline.

| Signal | Action |
|---|---|
| Prompt contains "explain", "how does", "what is", "compare", "summarize" | Answer inline, no plan mode |
| Prompt asks to add, modify, remove, rename, refactor, fix, implement | Plan mode is acceptable |
| Prompt says "just tell me" / "no plan needed" / "quick question" | Never enter plan mode |

### 8. Switch to a feature branch before writing on `main`

The `branch-protection.sh` hook blocked 5 stage/push attempts on
`main` across 5 sessions. The remediation is in
`.claude/rules/handling-blocked-hooks.md`: `git checkout -b
feature/<name>` before staging, or use an explicit refspec
(`git push origin main:feature/<name>`) to migrate local main commits.

Do **not** prefix the override env var inline — the hook ignores it
when the agent (rather than the human shell) sets it.

### 9. Skip self-review-request on `gh pr create`

Four sessions hit `HTTP 422: Review cannot be requested from pull
request author`. Before adding reviewers via
`gh pr edit --add-reviewer X`, check whether `X` is the current `gh
api user --jq .login` — and omit self.

### 10. Use the fully-qualified plugin agent ID

Three "Agent type 'X' not found" errors occurred when the bare name
was used (`security-audit`, `test`,
`github-actions-plugin:github-actions-inspection`). The correct form
is the namespaced `<plugin>:<agent>`, e.g.
`agents-plugin:security-audit`. When unsure, the available list is
shown in the error message itself — pick the closest match from
that list rather than retrying with another guess.
